## Trust Boundaries
| Boundary | Description |
|---|---|
| Git refs/paths → Node policy | Branch-controlled strings enter Git and path classification. |
| CLI → repository files | Bump commands may write canonical records. |

## STRIDE Threat Register
| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-01-01 | Tampering | refs/paths | high | mitigate | Reject absolute/traversal paths, normalize `/`, and use argument arrays. |
| T-01-02 | Elevation | child processes | high | mitigate | No shell interpolation; bounded Git subcommands only. |
| T-01-03 | Tampering | version writes | high | mitigate | Atomic writes are limited to VERSION and parsed manifest roots; tests assert no other diff. |
| T-01-04 | Disclosure | diagnostics | medium | mitigate | Print refs/paths and actionable errors only, never environment or file contents. |
| T-01-05 | Elevation/Disclosure | PR boundary | high | mitigate | Plan 01-09 supplies read-only PR permissions and no privileged trigger. |
| T-01-06 | Tampering | generated inventory | high | mitigate | Plan 01-07 enforces canonical JSON/report freshness. |
| T-01-SC | Tampering | npm installs | high | mitigate | This plan installs nothing; Plan 01-02 gates later exact installs. |
</threat_model>
