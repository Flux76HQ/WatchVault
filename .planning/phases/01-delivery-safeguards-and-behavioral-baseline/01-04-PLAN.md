<threat_model>
## Trust Boundaries
| Boundary | Description |
|---|---|
| Git staged index → hook | Contributor-controlled staged paths enter enforcement. |
| VERSION/build arg → image metadata | Build input crosses into public image/runtime metadata. |

## STRIDE Threat Register
| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-01-01 | Tampering | staged paths/refs | high | mitigate | Shared engine normalizes/rejects unsafe paths and hook delegates without parsing. |
| T-01-02 | Elevation | hook/setup commands | high | mitigate | Fixed argument arrays and POSIX exec; no shell interpolation. |
| T-01-03 | Spoofing | image/runtime/Compose version | high | mitigate | Required build arg, copied canonical file, adapter-derived Compose/CI input, drift tests, endpoint tests, and later tag CI check. |
| T-01-04 | Disclosure | image artifacts | high | mitigate | Copy VERSION only; no environment dumps or credentials. |
| T-01-05 | Elevation/Disclosure | PR workflow | high | mitigate | Plan 01-09 uses read-only PR permissions and excludes privileged triggers. |
| T-01-06 | Tampering | generated inventory | high | mitigate | Plan 01-07 checks canonical JSON/report byte equality. |
| T-01-SC | Tampering | dependencies | high | mitigate | Docker uses lock-preserving `npm ci`; installs follow Plan 01-02 approval. |
</threat_model>
