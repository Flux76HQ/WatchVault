<threat_model>
## Trust Boundaries
| Boundary | Description |
|---|---|
| Git refs/paths → policy tests | Untrusted refs and path strings enter process and classification logic. |
| Repository fixtures → generated evidence | Malformed paths, inventory data, and generated files may attempt traversal or conceal drift. |
| PR code → CI/artifacts | Fork-controlled code must never receive write privileges or real credentials. |

## STRIDE Threat Register
| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-01-01 | Tampering | Git refs/path fixtures | high | mitigate | Tests require normalized Git paths, reject absolute/`..`, and pass refs through argument arrays. |
| T-01-02 | Elevation | Shell execution | high | mitigate | Tests prohibit shell interpolation and exercise `spawnSync`/`execFileSync` argument arrays. |
| T-01-03 | Elevation/Disclosure | PR boundary | high | mitigate | Later CI contract requires `pull_request`, read-only permissions, and excludes `pull_request_target`. |
| T-01-04 | Information disclosure | Artifacts/fixtures | high | mitigate | Test fixtures are synthetic and later artifact checks exclude credentials/environment dumps. |
| T-01-05 | Tampering | Generated report | high | mitigate | Byte-freshness and canonical-JSON tests make generated-file tampering fail closed. |
| T-01-06 | Tampering | Dependencies | high | mitigate | Exact pins and the blocking legitimacy checkpoint precede installation. |
| T-01-07 | Spoofing | Docker/Compose build version | high | mitigate | Contracts require one parsed root VERSION adapter for local and CI builds and reject missing, malformed, conflicting, or hardcoded input. |
| T-01-SC | Tampering | npm installs | high | mitigate | No install occurs until Plan 01-02 approves every SUS package. |
</threat_model>
